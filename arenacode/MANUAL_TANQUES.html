1. Estrutura do Arquivo

Cada tanque vive em um arquivo .py dentro da pasta arenacode/tanks/. Para criar o seu, siga exatamente este modelo:

from core.tank import Tank    # Importa a classe base (obrigatório)
from core.action import Action # Importa todas as ações disponíveis (obrigatório)

class MeuTanque(Tank):          # Crie sua classe herdando de Tank
    def __init__(self):
        # Defina o nome e a cor do seu tanque aqui
        super().__init__(nome="MeuTanque", cor=(255, 100, 50))

    def programar(self):
        # TODA a lógica de comportamento fica aqui
        self.patrulhar([...])
        self.inimigo_encontrado([...])
        self.recebendo_ataque([...])
        self.fugir([...])

Regras importantes:
- O nome da classe (ex: MeuTanque) deve ser único no projeto.
- O nome dentro de super().__init__() é o que aparece na tela.
- A cor é uma tupla RGB: (Vermelho, Verde, Azul), cada valor de 0 a 255.
- Você obrigatoriamente deve implementar o método programar().

2. O Método programar()

O programar() é chamado uma única vez no início da batalha. É nele que você diz ao tanque como agir em cada situação.

Você não escreve if ou while dentro dele — você lista ações sequenciais para cada estado. O motor do jogo cuida de executar essas ações na ordem certa, no momento certo.

def programar(self):
    self.configurar_gatilhos(vida=30, municao=5, alerta=2.0)  # Opcional

    self.patrulhar([
        Action.MOVER_FRENTE(150),
        Action.GIRAR_DIREITA(90),
    ])

    self.inimigo_encontrado([
        Action.MIRAR,
        Action.ATIRAR_MEDIA,
    ])

    self.recebendo_ataque([
        Action.ESQUIVA_SMART,
        Action.MIRAR,
        Action.ATIRAR_FORTE,
    ])

    self.fugir([
        Action.MOVER_TRAS(100),
        Action.GIRAR_ESQUERDA(90),
    ])

Como funciona na prática:
A cada frame do jogo, o motor analisa a situação do tanque, decide em qual estado ele está (patrulhando, atacando, etc.) e executa a próxima ação da lista correspondente. Quando a lista termina, ela recomeça do início automaticamente (loop).

3. Os 4 Estados do Tanque

O tanque possui 4 estados que determinam qual lista de ações será executada. O motor troca de estado automaticamente conforme a situação do campo de batalha.

Prioridade dos Estados (do mais urgente ao menos urgente):

FUGIR  >  SOB ATAQUE  >  INIMIGO ENCONTRADO  >  PATRULHANDO

Isso significa: se o tanque estiver com vida baixa E com inimigos por perto, ele entra em FUGIR, não em INIMIGO ENCONTRADO.

🔵 Estado 1: patrulhar — "Nada acontecendo"

Quando ativa: Quando não há inimigos detectados, não está recebendo ataque e os recursos ainda estão acima do limite de fuga.

Para que serve: Movimentar o tanque pela arena enquanto procura inimigos. Um bom patrulhamento evita que o tanque fique parado num canto.

self.patrulhar([
    Action.MOVER_FRENTE(200),    # Anda 200 pixels para frente
    Action.GIRAR_DIREITA(90),    # Vira 90° à direita
    Action.VARREDURA_RADAR(360), # Gira 360° procurando inimigos
])

Dica: Use Action.VARREDURA_RADAR para sair do estado de patrulha mais rápido — ele avança automaticamente assim que detectar um inimigo.

🔴 Estado 2: inimigo_encontrado — "Tem alguém aqui!"

Quando ativa: Quando o sensor detecta um inimigo dentro do cone de visão frontal de 120° e a até 400 pixels de distância.

Para que serve: Definir o comportamento de combate. É aqui que você atira, persegue ou mantém distância.

self.inimigo_encontrado([
    Action.MIRAR,              # Gira até apontar para o inimigo
    Action.ATIRAR_MEDIA,       # Atira (custo médio de energia)
    Action.PERSEGUIR(100),     # Persegue o inimigo por 100 pixels
])

Atenção: O tanque só detecta inimigos em um cone de ±60° da sua frente e em até 400px. Se o inimigo estiver atrás ou muito longe, o tanque volta ao estado de patrulha.

🟡 Estado 3: recebendo_ataque — "Estou sendo atingido!"

Quando ativa: Toda vez que o tanque recebe dano, ele entra neste estado por um tempo configurável (padrão: 2 segundos). Mesmo que o inimigo saia do campo de visão, o estado permanece ativo durante esse tempo.

Para que serve: Reagir a um ataque — esquivar, revidar, se reposicionar.

self.recebendo_ataque([
    Action.ESQUIVA_SMART,     # Detecta balas e esquiva automaticamente
    Action.MIRAR,             # Aponta para o agressor
    Action.ATIRAR_FORTE,      # Contra-ataca com tiro pesado
    Action.MOVER_FRENTE(50),  # Avança um pouco para sair da linha de fogo
])

🟣 Estado 4: fugir — "Preciso sobreviver!"

Quando ativa: Quando a vida cai abaixo do limite configurado (padrão: 40 pontos) OU quando a munição fica abaixo do limite (padrão: 4 balas).

Para que serve: Sobrevivência. Neste estado, o tanque deve priorizar se afastar do perigo.

self.fugir([
    Action.MOVER_TRAS(80),     # Recua 80 pixels
    Action.GIRAR_ESQUERDA(90), # Muda de direção
    Action.MIRAR,              # Ainda tenta mirar
    Action.ATIRAR_FRACA,       # Atira fracamente (gasta pouca energia)
])

Importante: O estado fugir também é ativado com munição baixa. Isso faz sentido porque sem balas, continuar no combate é perigoso.

4. As Actions

Actions são os comandos que você passa para cada estado. Cada ação é executada até ser concluída, depois a próxima começa.

Existem dois jeitos de usar uma Action:

# Com valor explícito (você define quanto):
Action.MOVER_FRENTE(150)   # Move exatamente 150 pixels

# Sem valor (usa o padrão):
Action.MOVER_FRENTE        # Move 50 pixels (padrão)

📦 Tabela Completa de Actions

Action Valor Padrão O que faz
Action.MOVER_FRENTE(pixels) 50 px Move o tanque para frente a distância especificada
Action.MOVER_TRAS(pixels) 50 px Move o tanque para trás (velocidade reduzida à metade)
Action.GIRAR_DIREITA(graus) 45° Rotaciona o tanque no sentido horário
Action.GIRAR_ESQUERDA(graus) 45° Rotaciona o tanque no sentido anti-horário
Action.GIRAR_ALEATORIO(graus) aleatório Gira um ângulo aleatório entre 45° e 180°
Action.PARADO — Tanque fica parado por um momento
Action.MIRAR — Gira o tanque até apontar para o inimigo mais próximo (funciona em 360°)
Action.ATIRAR — Atira (alias de ATIRAR_MEDIA)
Action.ATIRAR_FRACA — Tiro fraco: pouco dano, pouca energia
Action.ATIRAR_MEDIA — Tiro médio: dano e energia moderados
Action.ATIRAR_FORTE — Tiro pesado: muito dano, consome muita energia
Action.ATIRAR_CERTEIRO — Só atira se o alvo estiver dentro de 5° de mira
Action.PERSEGUIR(pixels) 50 px Gira em direção ao inimigo e avança ao mesmo tempo
Action.MOVER_SEGURO(pixels) 50 px Move para frente mas para automaticamente se uma parede estiver a menos de 50px
Action.ESQUIVA_SMART — Detecta balas vindo de frente e esquiva automaticamente
Action.VARREDURA_RADAR(graus) 360° Gira até detectar um inimigo ou completar o ângulo especificado
Action.MANTER_DISTANCIA(pixels) 250 px Aproxima ou afasta do inimigo para manter a distância ideal

🔫 Detalhes dos Tipos de Tiro

Tiro Dano Custo de Energia Velocidade da Bala
ATIRAR_FRACA 3 pts 10 energia 7.0 (lenta)
ATIRAR_MEDIA / ATIRAR 6 pts 25 energia 11.0 (média)
ATIRAR_FORTE 12 pts 50 energia 16.0 (rápida)
ATIRAR_CERTEIRO 6 pts 25 energia 11.0 (média, mas só atira se mirando)

Regra de cooldown: Após cada tiro, o tanque aguarda 45 frames (~0,75 segundos) antes de poder atirar novamente, independente do tipo de tiro.

Recuperação de munição: A cada 3 acertos no inimigo, o tanque recupera 1 bala. Precisão é recompensada!

💡 Comportamentos Especiais Explicados

Action.MIRAR

Não atira — apenas gira o tanque até estar apontado para o inimigo mais próximo. Quando a mira estiver alinhada (erro < 3°), a ação é concluída e a próxima começa.

Busca em 360° — diferente do sensor, o MIRAR encontra o inimigo mesmo que esteja atrás do tanque.

self.inimigo_encontrado([
    Action.MIRAR,        # Primeiro alinha a mira...
    Action.ATIRAR_MEDIA, # ...depois atira
])

Action.PERSEGUIR(pixels)

Faz duas coisas ao mesmo tempo: gira em direção ao inimigo enquanto avança. Ideal para perseguições. Se colidir com algo, pula para a próxima ação.

Action.ATIRAR_CERTEIRO

Diferente de ATIRAR_MEDIA, esta ação só dispara se o inimigo estiver dentro de 5° da mira atual. Se não estiver alinhado, a ação é pulada sem gastar munição ou energia. Ótimo para não desperdiçar balas.

Action.ESQUIVA_SMART

Analisa os sensores em busca de balas vindo de frente (ângulo relativo < 30°). Se detectar perigo, interrompe tudo e executa uma sequência de emergência: gira 90° e avança 50px. Depois retorna ao fluxo normal.

Action.VARREDURA_RADAR(graus)

Gira o tanque varrendo a área em busca de inimigos. Avança para a próxima ação imediatamente se detectar um inimigo. Se girar todo o ângulo sem encontrar ninguém, avança normalmente.

Action.MOVER_SEGURO(pixels)

Igual ao MOVER_FRENTE, mas com um sensor de parede embutido: se a parede estiver a menos de 50px na frente, para e avança para a próxima ação. Previne colisões desnecessárias.

Action.MANTER_DISTANCIA(pixels)

Monitora continuamente a distância do inimigo mais próximo. Se estiver muito perto, recua. Se estiver muito longe, avança. Termina quando a distância estiver dentro de ±20px do alvo.

🔧 Comportamento em Colisão com Parede

Quando um tanque (MOVER_FRENTE, MOVER_TRAS ou PERSEGUIR) bate em uma parede, o motor automaticamente executa uma sequência de recuperação:

  1. Gira 90° para a direita
  2. Avança 60 pixels

Isso acontece antes de qualquer coisa que você programou — é um mecanismo de proteção para o tanque não ficar preso.

5. O Sistema de Sensores

O objeto Sensor representa o que o tanque enxerga ao redor. Ele é gerado automaticamente pelo motor a cada frame — você não precisa criar nem modificar ele.

Os sensores são usados internamente pelas Actions inteligentes (como ESQUIVA_SMART e VARREDURA_RADAR). Mas é importante entender o que cada um contém para saber quando as Actions vão reagir.

Campos do Sensor

# Lista de inimigos detectados no cone de visão (±60° frontal, até 400px)
sensor.inimigos_proximos  # Lista de dicionários, ordenada por distância

# Cada inimigo na lista tem:
# { 'distancia': float,        → Distância em pixels até o inimigo
#   'angulo_relativo': float,  → Ângulo em graus (positivo = esquerda, negativo = direita)
#   'vida': int }              → Vida atual do inimigo (0-100)

# Distância até a parede em cada direção (em pixels)
sensor.parede_frente      # Distância da parede à frente
sensor.parede_direita     # Distância da parede à direita
sensor.parede_esquerda    # Distância da parede à esquerda

# Informações próprias do tanque
sensor.propria_vida       # Sua vida atual (0-100)
sensor.propria_municao    # Sua munição atual

# Balas inimigas próximas (até 200px)
sensor.balas_proximas     # Lista de dicionários
# Cada bala:
# { 'distancia': float,       → Distância da bala até o tanque
#   'angulo_relativo': float } → Ângulo relativo da bala

Atenção ao cone de visão: O sensor de inimigos_proximos só detecta dentro de ±60° da frente e 400px de raio. Já ações como MIRAR e PERSEGUIR buscam inimigos em 360° (varredura total do mapa).

6. Configurando os Gatilhos

Os gatilhos definem os limites que fazem o tanque mudar de estado. Você os configura com configurar_gatilhos() dentro do programar().

self.configurar_gatilhos(
    vida=40,      # Entra em FUGIR quando vida <= 40 (padrão: 40)
    municao=4,    # Entra em FUGIR quando munição <= 4 (padrão: 4)
    alerta=2.0    # Fica em SOB_ATAQUE por 2 segundos após levar dano (padrão: 2.0)
)

Exemplos de estratégias com gatilhos:

Tanque Agressivo — foge só no limite extremo:

self.configurar_gatilhos(vida=15, municao=1, alerta=1.0)

Tanque Cauteloso — foge cedo e fica alerta por mais tempo:

self.configurar_gatilhos(vida=60, municao=8, alerta=4.0)

Tanque Kamikaze — nunca foge:

self.configurar_gatilhos(vida=0, municao=0, alerta=0.5)

7. Recursos do Tanque

Vida

  • Começa em 100 e vai até 0.
  • Quando chega a 0, o tanque é destruído.
  • Não se regenera durante a batalha.

Energia

  • Começa em 100 e é consumida ao atirar.
  • Regenera automaticamente à taxa de ~21 pontos por segundo.
  • Se não houver energia suficiente, o tiro não é disparado (a ação é pulada).

Munição

  • Começa em 30 balas.
  • Cada tiro consome 1 bala, independente do tipo.
  • Recuperação por acerto: A cada 3 tiros que acertam o inimigo, você ganha 1 bala de volta.
  • Quando acaba, o tanque não pode mais atirar.

8. Sistema de Pontuação

A pontuação é calculada ao final de cada rodada:

Evento Pontos
Vitória (único sobrevivente) +100
Sobrevivência (tempo esgotado com vida) +25
Destruído -50
Cada ponto de dano causado +2
Cada tiro que acerta +10
Cada tiro que erra -2
Cada inimigo destruído +50

Fórmula:

Total = Vitória/Sobrevivência/Destruído + (dano × 2) + (acertos × 10) + (erros × -2) + (destruições × 50)

Estratégia de pontuação: Acertar é mais importante que atirar muito. Um tiro que erra custa -2 pontos, enquanto um que acerta vale +10 + 2 por ponto de dano.

9. Exemplo Completo Comentado

from core.tank import Tank
from core.action import Action

class TanqueEstrategista(Tank):
    """
    Um tanque equilibrado que usa visão de radar,
    mantém distância e atira com precisão.
    """
    def __init__(self):
        # Nome que aparece na tela e cor em RGB
        super().__init__(nome="Estrategista", cor=(80, 200, 120))

    def programar(self):
        # Configura quando mudar de estado
        # Foge quando vida < 25 ou munição < 3
        # Fica alerta por 3 segundos após levar dano
        self.configurar_gatilhos(vida=25, municao=3, alerta=3.0)

        # PATRULHAMENTO: Varre a arena em zigue-zague
        # Sequência: anda → vira → anda → vira → repete
        self.patrulhar([
            Action.MOVER_FRENTE(200),    # Avança 200 pixels
            Action.GIRAR_DIREITA(90),    # Vira 90° à direita
            Action.VARREDURA_RADAR(360), # Faz uma varredura completa
            Action.MOVER_FRENTE(150),    # Avança mais 150 pixels
            Action.GIRAR_ESQUERDA(90),   # Vira 90° à esquerda
        ])

        # COMBATE: Mira, atira e mantém distância segura
        self.inimigo_encontrado([
            Action.MIRAR,                   # Alinha a mira no inimigo
            Action.ATIRAR_CERTEIRO,         # Só atira se estiver bem mirando
            Action.MANTER_DISTANCIA(180),   # Mantém 180px de distância
            Action.MIRAR,                   # Re-alinha (o inimigo pode ter se movido)
            Action.ATIRAR_MEDIA,            # Atira normalmente
        ])

        # SOB ATAQUE: Esquiva e revida
        self.recebendo_ataque([
            Action.ESQUIVA_SMART,       # Tenta esquivar de balas vindo de frente
            Action.MIRAR,               # Aponta para o agressor
            Action.ATIRAR_FORTE,        # Revida com força máxima
            Action.MOVER_FRENTE(80),    # Sai da posição para não virar alvo fácil
        ])

        # FUGA: Recua e ainda tenta atirar
        self.fugir([
            Action.MOVER_TRAS(100),      # Recua 100 pixels
            Action.GIRAR_ALEATORIO(90),  # Gira aleatoriamente para confundir
            Action.MIRAR,                # Ainda tenta mirar
            Action.ATIRAR_FRACA,         # Atira fraco para não gastar muita energia
        ])

10. Dicas Estratégicas

🎯 Precisão vale mais que volume de tiros

Cada erro custa -2 pontos e gasta sua munição. Use ATIRAR_CERTEIRO ou sempre MIRAR antes de atirar.

🔄 Sequências em loop

As listas de ações formam um loop contínuo. Pense em sequências que fazem sentido sendo repetidas. Por exemplo:

# Loop de combate eficiente:
[Action.MIRAR, Action.ATIRAR_MEDIA, Action.MANTER_DISTANCIA(150)]
# 1. Mira → 2. Atira → 3. Reposiciona → volta para 1. Mira...

🛡️ Não deixe o estado fugir vazio

Se o tanque entrar em fuga e não tiver ações definidas, ele ficará parado — um alvo fácil. Sempre coloque pelo menos um MOVER_TRAS na fuga.

📡 Use VARREDURA_RADAR no patrulhamento

Sem varredura, o tanque pode não detectar inimigos que estão fora do seu cone de visão frontal. A varredura garante que ele encontre inimigos mesmo ao redor.

⚡ Gerencie a energia

ATIRAR_FORTE gasta 50 de energia. Se você usar muitos seguidos, o tanque ficará sem energia para atirar. Alterne com ATIRAR_FRACA (10 de energia) para se recuperar.

🔢 Limite de 5 ações por estado

Cada estado aceita no máximo 5 ações. Se você colocar mais, as extras serão ignoradas.

11. Erros Comuns

❌ Esquecer de chamar super().__init__()

# ERRADO:
def __init__(self):
    self.nome = "Meu Tanque"  # Isso vai quebrar o jogo!

# CORRETO:
def __init__(self):
    super().__init__(nome="Meu Tanque", cor=(200, 50, 50))

❌ Esquecer de importar o Action

# ERRADO:
from core.tank import Tank  # Só isso não basta!

# CORRETO:
from core.tank import Tank
from core.action import Action  # Necessário para usar Action.MOVER_FRENTE etc.

❌ Tentar usar if/while dentro de programar()

O método programar() é executado uma só vez, antes da batalha começar. Você não pode colocar lógica condicional nele — as Actions cuidam das decisões durante a batalha.

# ERRADO (não funciona como esperado):
def programar(self):
    if self.vida < 50:     # self.vida aqui é sempre 100!
        self.fugir([...])

# CORRETO: use configurar_gatilhos() para definir quando fugir
def programar(self):
    self.configurar_gatilhos(vida=50)
    self.fugir([...])

❌ Passar o tipo de ação errado

# ERRADO (strings não funcionam):
self.patrulhar(["MOVER_FRENTE", "GIRAR_DIREITA"])

# CORRETO (use Action.):
self.patrulhar([Action.MOVER_FRENTE(150), Action.GIRAR_DIREITA(90)])
